Stock Cycle Count

Adds the capability to schedule cycle counts in a warehouse through different rules defined by the user

Cycle Count Rule

Create as much cycle count rules as you want. If warehouse is selected in 'Apply this rule in' then the rule will be applied on the selected warehouses.

Stock Warehouse

Configure the Warehouse:
Maximum Discrepancy Rate Threshold: Maximum Discrepancy Rate allowed for any product when doing an Inventory Adjustment.
Cycle Count Planning Horizon:Only the counts inside the horizon will be created.
Inventories for location accuracy calculation: Number of latest inventories used to calculate location accuracy.
Cycle Count Rules applied in this Warehouse: Add the rules that need to be applied on this warehouse

Stock Location

Configure the location:
Maximum Discrepancy Rate Threshold: Thresholds defined in Locations have preference over Warehouse's ones.
Disable Zero Confirmations: Define whether this location will trigger a zero-confirmation
Exclude from Cycle Count: Define whether the location is going to be cycle counted.
Inventory Accuracy: Automatically calculated based on the inventory adjustment

Cycle Count for Warehouse

CYCLE COUNT

When cycle count is confirmed an inventory adjustment related to that cycle count is created.

INVENTORY ADJUSTMENT

Cycle count is linked with the inventory adjustment. Thresholds defined in Warehouse is taken. Discrepancy and Discrepancy percent is calculated based on the real quantity of product which is entered by the user. Accuracy is calculated based on the Discrepancy percent of all products.

Cycle Count for Location

CYCLE COUNT

Cycle count is created for the location. When cycle count is confirmed an inventory adjustment related to that cycle count is created.

INVENTORY ADJUSTMENT

Cycle count is linked with the inventory adjustment. Thresholds defined in Location is taken.
